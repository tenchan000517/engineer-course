```

# 入力データ
タイトル: {{ $json.title }}
ツール: {{ $json.tools }}
リサーチポイント: {{ $json.research_points }}

# 最重要指示（絶対厳守）

1. 入力データの「タイトル」「ツール」は一切変更せず、そのまま使用すること
2. ツール名やモデル名が不明・最新でも、入力されたものをそのまま出力に使うこと
3. 入力データを「古い」「存在しない」「間違い」と判断して別の名前に置き換えることは絶対禁止
4. Web検索やリサーチは行わないこと。入力データのみを使用すること
5. ベンチマークスコアや具体的な数値は生成禁止。機能の特徴のみを説明すること

# 禁止事項（厳守）
- 入力データのツール名を変更・修正すること
- 想像や推測で数値・スコアを生成すること
- 絵文字の使用（一切禁止）
- 抽象的な表現（「便利」「凄い」「魔法」など）
- 過激な表現（「〜を殺す」「〜は終わり」など）

# ナレーション制約
1. 合計300〜350文字（読み上げ60秒相当）
2. 冒頭5秒で「まだ手作業でやってるの？」等の課題提起
3. ツールの具体的な機能名を含める（入力のresearch_pointsを活用）
4. 「これを使えば○○が一瞬で終わる」という時短メリットを明示
5. CTAは「コメント」「保存」への誘導のみ（プロフリンク禁止）

# スライド制約
- 各セットは3行、各行は全角14文字以内（厳守）
- 英数字は半角2文字=全角1文字で換算
- 3行は対になる要素か同じ属性で揃える
- ツール名は入力データのまま正確に記載

# サムネイル制約
- thumb_main: 8文字以内（疑問形または煽り）
- thumb_sub: 6文字以内×3行（改行区切り）

# キャプション制約
- 絵文字禁止
- 親しみやすい文体（〜ですよね、〜なんです）
- 見出し（▶）で3セクションに分ける
- 箇条書き（①②③）でポイント整理

# 出力形式（JSON）
以下のJSON形式で出力してください。他の文字は含めないでください。

{
  "narration_1": "（0:00-0:30のナレーション）",
  "narration_2": "（0:30-1:00のナレーション）",
  "thumb_main": "（8文字以内）",
  "thumb_sub": "（6文字以内）\n（6文字以内）\n（6文字以内）",
  "set_1": "（1行目）\n（2行目）\n（3行目）",
  "set_2": "（1行目）\n（2行目）\n（3行目）",
  "set_3": "（1行目）\n（2行目）\n（3行目）",
  "set_4": "（1行目）\n（2行目）\n（3行目）",
  "set_5": "（1行目）\n（2行目）\n（3行目）",
  "set_6": "（1行目）\n（2行目）\n（3行目）",
  "main_tool": "（入力のツールをそのまま記載）",
  "caption": "（キャプション全文）",
  "hashtags": ["#タグ1", "#タグ2", "#タグ3", "#タグ4", "#タグ5"]
}

# 時系列対応（スライドとナレーションの同期）
- Set 1 (0:00-0:05): フック - 手作業の無駄を指摘
- Set 2 (0:05-0:15): 課題提起 - 視聴者が抱える悩み
- Set 3 (0:15-0:25): 解決策 - ツールの紹介
- Set 4 (0:25-0:35): 仕組み・機能 - なぜ時短できるか
- Set 5 (0:35-0:45): ベネフィット - 具体的なメリット
- Set 6 (0:45-0:55): 結論 - 使うべき理由
- CTA (0:55-1:00): 固定画像（生成不要）
```
